# High-Risk AI Technical Documentation (Structure based on Annex IV)

**Source:** Activity_3_3


*(This serves as a master table of contents for the full documentation package. Each item should be a separate document or section.)*

| Section | Title | Description | Owner |
| :--- | :--- | :--- | :--- |
| **1** | **General Description** | High-level overview, intended purpose, users, hardware, and software dependencies. | System Owner |
| **2** | **Instructions for Use (IFU)** | A standalone, user-facing document. (See template below). | Tech Writer |
| **3** | **System Architecture** | Detailed description of the AI system's architecture, components, and data flows. | Architect |
| **4** | **Development & Training** | Detailed information on the design, training, validation, and testing procedures and tools used. | Data Scientist |
| **5** | **Data Governance** | The completed **Data Sheets** for all training, validation, and testing datasets (from Activity 3.2). | Data Gov. Officer |
| **6** | **Risk Management** | The complete, up-to-date **AI Risk Register** for the system (from Activity 3.1). | System Owner |
| **7** | **Performance & Accuracy** | Detailed description of the system's performance metrics, accuracy levels, and robustness against cybersecurity risks. | QA Lead |
| **8** | **Human Oversight** | Description of the measures implemented to enable human oversight (from Activity 3.5). | UX/Product |
| **9** | **Post-Market Monitoring** | The plan for post-market monitoring of the system (from Activity 3.10). | System Owner |
| **10** | **Declaration of Conformity** | The formal, signed EU Declaration of Conformity (from Activity 3.8). | Legal/Compliance |
